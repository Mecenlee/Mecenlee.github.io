算法笔记

2023/03/22

1. 代码规范

变量命名：

规范

  • 大小驼峰:Java

  • 下划线:C

  • 匈牙利

小驼峰：第一个单词首字母小写后面其他单词都首字母大写theNum

大驼峰：第一个单词也大写TheNum

下划线：the_num

意义

空格

操作符左右要有空格

分隔符前一位没有空格，后一位有

大括号和函数同一行，同时一个空格隔开

while if for 后都有一个空格

2. 刷题用不用库函数

如果题目关键的部分直接用库函数就可以解决，建议不要使用库函数。

如果库函数仅仅是 解题过程中的一小部分，并且你已经很清楚这个库函数的内部实现原理的话，那么直接用库函数。

3. 算法的ACM模式

力扣上是直接写逻辑的，但是面试经常是直接牛客上啥也没有就去写。

力扣是直接搞算法逻辑的。

ACM就是记事本写算法那种。

主要是引包和一些输入输出，比如一些数据结构的构建。

4. 程序员写文档

MarkDown

专注写作，高效率。

富文本点来点去很麻烦，在写作的时候。

方便记笔记，更快速。

几个#就是几级主题

+/-表示无序列表

1.2.直接表示有序列表

链接：[名字](网址)

图片：![名字](图片地址)

还支持Web前端，html，CSS标签

软件：

一些网站

win：Typora（I am）

mac：macdown

6. 程序员简历

  1. 别花里胡哨，简单直接一点。

  2. 一张纸就行。

  3. 技术简历就不用来“我锻炼了组织能力了。”

  4. 别写精通XX语言

    精通 > 熟悉 > 掌握 > 了解

    简历上写了熟悉那门语言，基本就必考这个。

    一般编程语言只会考察一门。

  5. 拿不准的东西别写简历上。别写“代码行数10w+”

  6. 项目：难点 + 贡献

    做项目的时候要保证对难点的敏感程度。

    花了很久解决的东西一定要记下来形成博客。

    一般你难点写的什么就问你什么。

  7. 这样被动就成了主动。真正好的简历是 当同学们把自己的简历递给面试官的时候，基本都知道面试官看着简历都会问什么问题，然后将面试官的引导到自己最熟悉的领域，这样大家才会占有主动权。

  8. 勤劳输出博客 + github

  9. 专业技能要写的具体且有针对诱导性：熟悉计算机网络，熟悉TCP三次握手，四次挥手，流量控制，拥塞控制等手段。

  10. 项目经历： https://code-thinking-1253855093.file.myqcloud.com/pics/20220308225933.png

    项目描述，主要工作，项目难点，个人收获

    一般都是问后面两个。

7. 大厂面试流程

  • 一面机试：选择/编程
  • 二面基础算法面：算法
  • 三面综合技术面：编程语言+计算机基础+项目
  • 四面技术leader面：解决问题快速学习
  • 最后hr面：企业文化，职业发展，offer选择，薪资待遇

机试：选择》计网，操作系统，数据库，编程语言 牛客有

基础算法：白纸写基础算法。

Java/C++

尤其注意基本机构定义和白纸代码训练。

综合技术：

编程语言：Java -》 Kotlin

计算机基础：计网 - > 操作系统

项目：技术原理 + 技术深度 + 应变能力

Leaders面：

解决问题能力：

  • 学习/项目中遇到那些挑战，怎么解决的。

快速学习能力：

  • 如何快速学习一门新的技术和语言。

HR面

  • 为什么选择我么公司？技术氛围，职业发展，公司潜力
  • 职业规划，技术方面讲
  • 是否接受加班
  • 坚持最长的一件事情
  • 工资XX能接受嘛？

9. 时间复杂度

O(n):是最坏情况

O(1)常数阶 < O(logn)对数阶 < O(n)线性阶 < O(nlogn)线性对数阶 < O(n^2)平方阶 < O(n^3)立方阶 < O(2^n)指数阶
