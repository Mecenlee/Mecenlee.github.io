new 一个对象的过程

Animal a1 = new Animal()经历的过程1.在⽅法区寻找Animal类信息2.如果没有这个类，类加载器将当前这个类的字节码⽂件加载到内存中3.在堆中申请内存空间4.当前的创建的对象的地址会放在操作数栈中0x1005.在当前线程栈->⽅法栈帧->局部变量区域申请内存空间给a16.从操作数栈中弹出顶部操作数0x100赋值给a1第2步有个问题:这个类在内存中的存在形式是什么？类对象

为什么有构造方法

构造⽅法：为什么要有构造⽅法？

1.默认的⽆惨构造⽅法类本身不知道如何创建⾃⼰的对象，对象本身是由Object(){}来构建再由当前这个类来初始化那个对象

2.重载的有参构造⽅法：a.为了⽅便给初始化值b.在创建对象的同时给予某些需要的数据(时机)

单例设计模式

如何巧妙使⽤private来私有化构造⽅法给外部提供⼀个单例对象(整个程序运⾏中这个类只有⼀个对象)

1.私有化构造⽅法->外部⽆法创建这个类的对象private Animal(){}

2.提供⼀个静态的成员属性->静态变量只会初始化⼀次public static Animal instance = new Animal();

3.给外部提供⼀个静态⽅法->获取这个对象//注意:这⾥没考虑内存消耗和线程安全public static Animal getInstance(){return instance;}

4.外部获取这个对象Animal.getInstance();系统类库中以下开头的⽅法通常都是提供单例instanceshareddefault
