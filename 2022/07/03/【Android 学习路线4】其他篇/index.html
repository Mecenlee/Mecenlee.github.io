#【Android 学习路线4】其他篇

  1. Android 开发工具；
  2. 版本控制；
  3. Android 未来发展方向；
  4. 职场智慧；

开发工具

这个其实没啥好说的，基本都2023年了。

开发 Android 不会还有人在用 Eclipse 吧。

现在用的是 Android Studio(AS)

这个其实就是 IDEA 换层皮，哈哈哈。

就是要注意的是，AS 最好更到最新版。

不然有些新的 Android 一些东西用不了，比如 Compose 啥的。

我还在用 花栗鼠 版本。挺好。

这个 AS 开发工具的一些用法，那太多了。

估计都能出套系列课了。

还是那个大原则：遇到啥就学啥。

从怎么创建项目 -> 运行 -> 插件 -> Git -> 模板。

遇到了再去搜下就中了。

这个 AS 用习惯了在code的时候很爽的，这里推荐一本书吧。

不过推荐是你入门之后再去看看，大概你入职之后。

徐宜生大佬(我们都叫他“医生”，hhh)的《群英传》

这书有两本，第一本是讲 Android 开发技巧的，然后第二个系列《神兵利器》那个是专门讲 AS 的一些奇妙操作。

写的不错，就是有点上了年纪(doge)。随手看看也有不少收获。

后面有人催我，我也出几期视频好了。(不过没人催，还是懒狗鸽哈哈哈)

版本控制

这个版本控制说的好官话啊，其实就是 Git 那些东西。

对应三个网上放代码的云仓库。

Gitee，GitHub，GitLab。

咋理解呢，就是把你在电脑上写的 代码 通过 Git 这个工具，存在网上的那三个仓库。

你可以把代码仓库理解成你存资料的百度网盘，阿里云盘，啥的。

就是他是专门放代码的，然后要通过 Git 搞上去。

咋有三个呢，我介绍一下吧。

Github 这是最正宗的，最主流的那个。不过它是国外的服务器，你在中国如果不翻墙，就好慢。

推本地代码上去，总是失败。

Gitee 就是国内模仿 Github 的。这个不用翻墙，而且全是中文，符合中文开发者习惯。

就是因为 嗯哈 一些不可抗力，限制很多。

你把Gitee 理解为 GitHub 国内阉割版就可以啦。

然后这个 GitLab 啊，他是局部的。啥意思。

前面无论是 Github，还是 Gitee，他是面向所有人，不管你是谁，你都可以使用。

但是GitLab它是每个公司，每个组织自己建立一个自家的网上代码仓库。

就只给自己员工用。

你可以这么理解，GitHub 和 Gitee，是流量。不管你是哪里的电话卡，无论你走到哪，都有网用。谁都可以用。

然后这个 GitLab 就是你家里的 WiFi，只能你家里人连上用，只能在你家用。不是你家人，出了你家门就莫得了。

我们一个自己放代码是 GitHub 或者 Gitee。

我一般是 GitHub，因为叔会“科学上网”；hhh。

然后公司就用的是GitLab,每个人发一个账号，数据在公司自己的服务器上。

也有一些别扭的国企不是 GitLab，而是自己搭了个类似的小众平台。

本质和 GitLab 一样。

大家初学者，一开始直接用

Gitee，然后都熟悉了会翻墙了。

就用 GitHUb。

这两个会用了，公司要求 GitLab 其实一样的。

怎么用 Git 和Gitee网上很多教学视频了。自己去看看。

对了，他们都 Git 用命令行。说程序员要有这个范，我觉得~看个人喜欢。没必要非要用命令行装个B，点点点多快乐啊。(我是懒狗，勿cue)

直接图形化点点，多方便。哈哈哈，你喜欢那种就选那种吧

Git命令行和图形化差不多。

我不是运维，好久没用Linux我懒得都是用图形化了。hhhh

谁让我是前端捏，我不仅菜我还懒。hhh

Andrid 发展方向

其实啊，我不想写这个。因为我面向得准备搞 Android 的小白。

初学者，专心入了门。就可以了，说太多高阶的东西，往往是让大家迷了眼睛。也容易压力给到你那边。

所以这个板块，大家就当看个乐子。有个基本认识就中。别深究。

起码在你入职中大厂3年之前，你是完全不需要考虑的。

Android 未来深入的方向啊，我觉得机会很大的，可行性比较高的有下面4个：

  1. 车载系统；
  2. 跨端/大前端
  3. Framework框架
  4. 音视频

车载系统：

这个就是那个汽车，驾驶位置旁边不有个显示屏嘛？

你点进去，看下设置：Android9.0

熟不熟悉。

对吧，不就是 Android。一样的东西。

为啥说他是大风口呢。

2022的IT求职，那不全是迪子哥(比亚迪)救起来的。秋招滴神。

现在无论大小公司都想来智能汽车插一脚，都在炒概念。

国内的汽车突然对他的中控那个系统需求很大，然后那个就是Android。

比如特斯拉那个车载，做的真心优秀。

现在很多 Android网上培训机构已经开始卖车载系统的课了。说明这个风口很有盼头的。

跨端：

这个其实是19年搞起来的。他的理念就一个。把所有前端合在一起。就是写Web的前端，写Android的 ,写IOS的，小程序，PC桌面端

后端就招一个就中了，不管是Java后端还是C++，go后端。一个人，后面就全拜托它了。

资本家就想：啊，这个前端分这么多啊，不就显示个画面咩。

然后就想着，能不能发明个，写一个代码。然后直接生成上面网页，安卓手机，苹果手机，小程序，PC应用这么多。

这就是跨端的概念和由来。

因为很贴合国内资本家的这个缩本增效，所以前几年很火。

跨端技术啊，有下面几个：Flutter，RN，uniapp，H5，还有腾讯阿里美团自己也出了点跨端的东西。

用的比较多是 Flutter 和 RN 。前面是我们 移动端 -> 跨端喜欢的。RN是Web前端 -> 跨端喜欢的。

其实吧，说句良心话。这个目前做的是不行的。

为啥呢？一开始大家都是用原生，后来这个概念起来了。APP就开始一半原生，一般跨端技术。

比如阿里的咸鱼APP，就是一般Android原生，一半 Flutter。

但是现在出现个啥子情况呢？就是国内移动流量见顶了。上到70/80老人，下到小学一年级小学生都玩手机。

手机普及率99%+了。

所以市面是大家用的APP基本都被几个大厂给包了，而且他们发现用 跨端，APP真的会卡，然后影响用户体验。

所以咸鱼现在又开始去跨端，开始用原生去重构了。

属于是：

原生 -> 跨端/原生 -> 原生

这点你从字节跳动每年招的 移动端的人数就看得出来。

大家发现 跨端不怎么走的通了。

不过也有企业还在跨呢，比如美团，我一学妹Android进去，现在RN开发。hhh

Framework框架又是啥啊？

咋又是Android风口呢？

哎呀，我最烦这些专业术语。装什么装嘛。我解释的累死了。

这玩意就是你手机的系统。

比如小米的MIUI，华为吹的鸿蒙。还有苹果的IOS。

这些都叫手机操作系统，每个手机品牌都喜欢把Android原生系统拿过来，然后在上面改点加点东西。

然后就说这是我们家定制的啥啥系统。

这个加点，改点东西的过程就是我们Android人去做的Framework框架。

如果有朋友玩酷安的话，就知道这就是做的rom。

每年各大手机厂商，啥OPPO，vivo，荣耀啊，就会招Android去给他们做系统。这就是这个方向。

音视频

这个多亏抖音啊，开始的时候是做直播，后来就是抖音。

这种快速大量的流媒体，其实对手机APP的要求是很苛刻的。

最容易出问题，出Bug。

这方面是做Android 挖的最深的。但是一般要学C++。

所以一般人不会做，做了就成了专家。

且这个不担心35危机，这个领域属于越老越吃香。

但是C++要求很高

上面讲的是 走 Android 自己的路。

其实还可以走和Android相关的路。

我指两个最适合的：Java后端，Web前端。

注意限定词语。后端是Java

为啥呢，因为现在做后端还有C++，go

如果你开发Android用的不是Kotlin而是Java的话。那你的Java基础其实是可以的。

基本补上Web和框架也能做，不过Java后端极其的卷，无线逼近于算法。

好处是岗位一下子翻了好多好多倍，且35危机会稍微好点，晋升天花板也好很多。

还有一个就是Web前端，因为无论是Web前端写网页还是Android写APP。其实都是写页面，本质上差不多。

上面说的大前端，他两就在一起。代码思维是差不多的。

有人可能说啊这个Java后端，和Web前端怎么可能。和Android差太多了。

我个师兄本来Android都做了2年了，然后被京东Java后端挖过去了。含泪弃暗投明(doge，一般是Java来Android，他反向上大分)

有个师妹上文说了，也是写Android，然后后来在美团的时候，从Android -> RN -> Web 前端。现在成了前端了。

这种例子不少，也不算的是一条路子。

害，程序员就一块砖，哪里需要往哪里搬。

少年郎，所以说，应试成绩不重要，但学习能力很重要嘛。

公司，社会看重的也是这个。

职场智慧

啊，写的好多啊。写了我一个晚上。

怎么说呢，虽然程序员是全中国，可以说最最看重实力的一个行业。

在这里你可以不会敬烟，不会酒桌文化，不用太懂眉眼高低，不用给老大送礼啥的。

还都搞个啥扁平化管理。可以说非常的现代化，非常的接近新时代工作的，年轻人喜欢的。

但是，我要给你泼冷水了。

有人的地方就有政治！

职场就不像学校那么单纯了。

这方面没办法几行字随随便便就说清楚。

随便点几个吧：

  1. 你的输出要兑换成KPI
  2. 你的东西最好要让老大知道
  3. 要懂向上管理，合理化领导的预期

哎呀，这种东西不太好说。

这样罗列了，你也看不懂。

举个例子吧。

比如领导交给你一个任务，说7天做完。

你自己明明可以1天搞好的，很轻松。

你缺还要装出一副，啊领导这个这个需求这个技术需要怎么怎么，需要那个地方培养培养，需要磨合啥啥啥。

起码需要14天。

还要说自己克服克服。

然后老大等不了，折了个中，给了你10天。

你还是不能立马就做，你要假模假样的。

先做完90%，然后就开始摸鱼。磨羊工。

每天就做一点点，到最后ddl的时候在去补上开始测试。

可能有时候没这么好啊，我的意思我想你这么聪明应该看的懂。

别用力太猛了，有些老大他也懂技术。

但是不知道你什么水平，你可以开始藏拙。然后每次都进步一点点。这样就不用得 D 绩效。

哈哈哈哈
