入门算法课推荐(Android)

”做 Android ，有没有推荐的算法入门课？免费！“

OK，这篇文章解决一下这个问题：

  1. 推荐一门【免费】的，最适合 Androider 的算法入门课；
  2. 看这个视频课的时候的各种细节；

事先说明，全片无广且免费。

同时详细步骤和细节注意都具体到保姆级,像写的这么细的，全网独一份。

好，我们开始。

一，Androider用什么编程语言去学算法？

一般有两个选择：Kotlin VS Java

我给答案是：Java

为什么呢？

看下面：

  1. 讲算法的教学资源，Java很多且丰富，Kotlin几乎没有；
  2. Androider Java基础也还重要，正好借此机会复习巩固 Java 基础；

虽然我们 Androider 开发主流是 Kotlin。用的也舒服也多。

但是

Kotlin的学习资料太少了，市面上没有一家用 Kotlin 讲算法的教学视频，连力扣上用 Kotlin 写的题解都不多。

但是 Java 树大根深，网上用 Java 讲算法的有且还有不要钱的质量不错的。

同时Androider 趁机巩固巩固 Java 也是很好的。

更多 Android Boy 对于编程语言的把握，这篇文章写的很明白了。

没看的朋友记得去补课：

点击直达：【Android学习路线01】编程语言 (安卓入门火山君)

网址：https://mp.weixin.qq.com/s/j99duaal7boHiHUoUW1G9A

二，Java算法入门资源推荐

不废话。

B站搜索：“算法 Java”

然后那个尚硅谷的就是(下图标记了)。

这是我考察了很久找到的，用Java讲的算法入门质量很高且免费的一套课。

是咱们的老朋友：韩顺平老师讲的。

点击直达：【免费高质量】B站韩顺平Java算法入门)

具体网址：https://www.bilibili.com/video/BV1E4411H73v

配套资源：数据结构韩顺平Java https://www.aliyundrive.com/s/NeqoxGgBWnn

三，为什么是这个？

本着”省钱且高质“的原则，我测评了网上几乎所有用 Java 讲 算法的免费教学视频。

最后从 Android 新手 免费 高质量 细致 的角度多方面考究，上面老韩的是最后解。

讲的细致全面，基本能覆盖的都有了。虽然有些小坑，但是我当初也是拿这个学的。接近两百集我都替大家过了一遍。

具体的哪里有坑，以及遇到的详细应对措施我都写在后面了。

四，具体“食用”指南(细节拉满)

4.1 怎么看

老师讲课的逻辑是：场景引入，思路分析，代码实现，课后习题。

我推荐这样：

  1. 看到思路分析，后面的就尝试全部自己完成。
  2. 写好了之后不看视频，直接看老师的笔记看下有啥他写了然后我没有优化的。补上。
  3. 然后思维导图把代码插入附件，并写下总结就走了。
  4. 然后继续下一个知识点

如果不会再去我老老实实看老师的代码实现，看懂后再自己实现一下并总结。

4.2 小细节(避坑)

  1. 递归是有点难的，主要难在理解这个“递归思想”。精髓我觉得是“整体思想”，跳出来，别卡在具体哪一行代码的牛角尖里。

  2. 韩老的下图三个排序讲的不清楚，这一段建议去隔壁黑马的Java算法学。那边思路讲的清晰一点。

    地址：https://www.bilibili.com/video/BV1iJ411E7xW

  3. 赫夫曼编码哪里是有点复杂度的，且不好验证自己有没有对。

    尤其是位运算和二进制的一些东西正码反码补码的一些计算机基础知识。

    如果你学Java用的我上次推荐德老韩那套JavaSe还好点，他哪里讲了这些东西。

    要是没有，不会二进制啥的也没有关系。别担心，放过去。不影响的。

    把那个解压方法当成一个黑盒子方法就行了，这个不影响算法的。是一些计算机基础的知识。

  4. dfs，bfs那个地方。推荐去隔壁黑马，链接在上面

4.3 其他补充

  1. 直接从 P6 开始看；

  2. 如果基础很好可以1.5倍速，个人建议不开倍速，给自己留点思考时间；

  3. 如果时间够，每天就看一个小节，然后老老实实的完成老师布置的所有思考题；

  4. 建议别开弹幕，影响心态。

  5. 学习计划安排不要“每天学几个章节，几p”，因为不同地方耗费时间不一样。

    有些章节内容少，有些多。有些章节很难，有些比较容易。

    我建议别太关注进度，用每天学一下午/一晚上。这种养成固定习惯。

五，个人总结

  1. 韩那个讲的很底层 思路很好 但老师有时候表述的 嗯 就一种”茶壶煮饺子——有货倒不出“的感觉。

    建议看韩的思路分析，再自己看他给的代码，这样效率快。

  2. 老韩讲的有一股C的味道，没有Java的那种面向对象的感觉。其实这样更好。

  3. 韩老的逻辑和代码能力 max，语言表达能力还有不小提升空间。肉眼debug，如果不是剧本的话。

隔壁黑马：这个更加的Java味道，面向对象。

但算法漏洞相对多点，且这个讲的很浅，代码有很多问题 适合救急。

我默认大家都是想好好学，从长远的角度来说，最后还是推了老韩的Java算法课了。
