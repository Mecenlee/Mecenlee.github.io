开发Android，我这新手是学Java还是Kotlin啊？

我们先来剖析一下这个问题。然后最后的解决方案我放在最后的总结部分。

情况：

就是18年之前，开发Android APP。用的主流就是 Java。

但是啊，在19年之后，这个 Android 亲爹跟 Java 亲爹两个掐起来了。

Android 就找了个 Kotlin，并开始主推。

有人可能没看懂我上面这段话的意思。

意思就是现在不是你选不选谁开发 Android 的问题。

是人家官方要“去Java，推Kotlin”。

如果懂的同学就知道了。

这叫政策，这是趋势，这是一个什么，官方表态。

包括你现在去看什么 Android 官网，他的一些官方文档和一些新的库。

基本全是用 Kotlin 写的，官方的大手在推动这件事情况的发生。

以后写 Android ，那是必定 Kotlin 的。没有回头草，没有啥不确定性的。

有的就是这个 Android 开发语言的迭代，谷歌什么时候推进完全。

那目前2023市场上搞 Android 是个啥情况呢。

我直说，18年以及以前。还是 Java 为主。

23年，基本主流是 Kotlin + jetpack 组件化了。

项目组撸新项目，都是 Kotlin First。

我问了下其他几个厂，出现 用 Java 写 Android的。

只有维护旧的项目。

当然，这种语言层面的迭代在不同公司是有快慢之分的。

比如某国企，他就是用 Java 写 Android。

额，不过。他们年前已经砍掉了移动端，直接用 前端 H5来包壳了。

哈哈哈

所以总结一下：

2023 搞 Android，上Kotlin是必须的。甚至大势所趋。

主流原生开发70%都是用的这个。

只有维护老的项目或者一些比较保守的公司会 Java。

如果你是刚接触 Android，我建议你直接上 Kotlin。

因为这里有一个时间差，今年2023年，是Kotlin70%。你是大一大二。

那等你几年后出来找工作，25/26届的时候呢？那时候估计Kotlin就基本完成平替了。

19-> 21 Kotlin 逐步取代 Java

21 -> 23 Kotlin成为主流

那23 -> 25呢？

别小看 Android 官方的态度。

就像在中国不要逆着政策发展一样，技术领域也有“官方”旨意。

好了，我是火山君。

一个聚焦于 Android 开发入门的老哥。
