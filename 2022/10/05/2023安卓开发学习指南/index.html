2023安卓开发学习指南

写这篇文章之前还专门去全网收罗了一下安卓开发相关的学习路线，不出所料，果真诸神黄昏。

无所谓，我会出手。（淦，发癫了）

废话不多说，加入收藏，你会用得到。

这篇《2023安卓开发学习指南》我把它分成了4个部分：

  1. 编程语言；
  2. Android；
  3. 计算机基础；
  4. 其他补充；

具体的逻辑架构可以看下面这张思维导图，本篇推文只是它的总体大纲。导图的内容更全乎详细方便，需要后台发送关键字：2023安卓开发学习指南 自取。记得给我点赞当辛苦费。

后面会做个视频细细讲解这张图，大家多催催我(越多人催，我更有动力去更快的更给大家)，到时候做好了，连接也会放在这个地方。

后面有什么变化会及时优化这篇指南(只要我还在搞安卓)，其他的需要补充的大家直接后台发我，后面我添加到第4部分；

那，正式开始。

一，编程语言

先后顺序就是重要程度，百分比是指该语言占编程语言学习的占比。

1. Kotlin(65%)

搞安卓开发，最重要的编程语言就是Kotlin。

建议：2023年搞安卓开发，Kotlin是重中之重，建议当成自己的最主力编程语言。

推荐资源：

  • 入门：《安卓开发入门课》中的 Kotlin 部分；
  • 进阶：Kotlin官方文档中文版

程度：熟练拉满/对应王者英雄熟练度的大红。

建议花费精力：65%

2. Java(35%)

怎么说，往前面数10年，开发安卓App就是用Java。但现在他的位置被Kotlin取代了，但是还是建议认真好好学下。

Why？

  1. 旧项目要。因为其实大家去工作，你不知道是跟着重新开发新的App项目，还是去维护旧的App项目。都有可能。旧的项目一般就是用Java写的，如果没重构成Kotlin的话。
  2. 学算法要。Android Boy 后面学数据结构和算法要用Java，算法很少有 Kotlin 版本的资料，但是 Java 版的就很多；
  3. 退路需要。Java的路子是比Kotlin广的，Kotlin就写安卓App有点局限，Java能做后端，大数据，开发安卓APP。是中国工业第一大编程语言，学好这玩意，真的性价比贼高且有必要。

所以我建议大家的第二语言就选这个Java，起码得化35%的精力去认真学它。

程度：认真掌握，大概王者英雄熟练度的小红那个程度，懂我意思吧

推荐学习资源：

入门：

  • [视频]B站韩顺平Java
  • [书]《Head First Java》

入门两者任选其一。

深入：

  • [书]《Java核心技术卷一》&《Java核心技术卷二》

建议：先看韩老师的视频然后看深入推荐的两本书。（《Java核心技术卷二》中和界面编程相关的不用学，放心跳过。）

再重复一遍：把Java当自己的第二语言，仅此于Kotlin。

今年有个朋友就是本来搞安卓APP开发的，在字节做 Android 做了有段时间了。后面被京东拉去做Java后端了。哈哈哈哈~一般都是Java后端调剂过来做安卓开发，这个哥反向上大分。

3. C(5%)

C语言是所有计算机科班大学的第一节编程课，虽然我们工作几乎不会用这玩意，但是当个开胃小菜学一下也行。

建议：时间很多就学的玩一下，熟悉熟悉。时间紧张，放心丢，没事。

重要性占比：5%

程度：简单了解，大概王者英雄熟练度从白板到绿色就可以

推荐资源：MOOC中国大学浙大翁恺的两门C语言课(进阶课中和图像相关都不学)

这个学到了解一些编程语言的基础概念就行：

比如

  1. 变量常量是啥？
  2. 数据类型有哪些？
  3. 运算符有哪些？
  4. 分支选择是个什么东西？
  5. 循环怎么写？
  6. 函数是啥，为啥这样好？
  7. 指针(C特色，有点难，没学会不碍事)

上面这些就是一门编程语言的基础构成，任何新的编程语言本质就是：上面那些 + 自己的一点特色。

说实话，C语言是最难学的，但又出现在大学刚来第一次接触编程就这玩意儿，真的很容易打击人。

其实除非是做嵌入式，不然C基本极少用，我个人的建议是：当个开胃小菜，借着了解一下编程，培养点程序的思想就行了。学不会也不用有啥子压力，反正重要性不是很那啥。

4.总结

4.1 然后具体编程语言怎么安排学的顺序呢？

个人推荐这样：

  • 急着上手(转来Android的)：问下leader，项目用Kotlin就学Kotlin，用Java就用Java。不确定就Kotlin，后面需要那个就学那个。

  • 有点时间但不充裕(大二/研二)：先上Kotlin，然后还有时间学Java，C不管。

  • 时间还早(大一/研一)：大一上学下C了解下，然后大一下&大二上学Java，大二下之后全部精力投Kotlin。

插一嘴：第3个是 Android Boy 最佳的一个编程语言学习节奏。

一般大学也是在大一开门C，然后大二开Java，但是Kotlin就没有。

哪怕开了啥移动开发的课，也是用Java写安卓开发。

所以按这个安排既是一个很好的学习频率，也能照顾学校的节奏。一举两得。

4.2 学语言时间精力分配

如果你真认定了走安卓开发，那就Kotlin70%Java30%。要是你是说就当安卓开发当个跳板，后面第二部有其他打算。那我建议KotlinJava各自50%。

你要是刚上大学本科或者研究生，想成为 Android Boy。那C5%，Java35%，Kotlin65%是一个很优雅的比例。

4.3 Java 进阶资源推荐：

其实按我推荐的Java学到这里就可以了，如果你尚有余力，想更加精进：

  • [书]《Java 编程思想》
  • [书]《Effective Java》
  • [书]《深入理解Java虚拟机》

4.4 学校还有其他语言，怎么处理？

省流：主要精力按上面给的分配学 Kotlin Java 啥的。学校的就期末前看看就能过了。

理由：

  1. 按照上面这样学编程语言，其实功底很扎实了。后面学一门语言很快，期末随便给一天就能看个7788；
  2. 大学里的编程语言课考试 很水。选择+填空+看流程图出结果+手写代码。80%的分都是背书，考的都是理论且浅。(放心，只要你真的认真去自学编程了，其实学校的编程相关专业课就洒洒水。亲身举例)

扯多了，还是那句话。我们就掌握好 Kotlin 和 Java，时间多学下 C。就好了，其他的学校有考试就考前看一下就可以了，不用专门花时间去学。

二，Android

1. 入门

重点：

  1. 四大组件。

    Activity，Service，ContentProvider，Broadcast Receiver

  2. Fragment

  3. 基础UI控件

    文本框，输入框，图片，按钮……

  4. 多媒体。

    相机，音乐，视频。

  5. 网络

资源推荐：

  • 《Android 开发入门课》中的 Android 部分
  • Android 官方文档

2. 进阶

重点：

  1. Material Design
  2. JetPack 组件
  3. Android Studio
  4. Android 部分源码
  5. ART

推荐资源

a. 书：《Android 开发艺术探索》

b. 书：《群英传》

c. 书：《Android源码设计模式解析与实战》(对应重点4)

d. 书：《深入理解 Android 虚拟机 ART》(对应重点5)

其实上面的重点，咱一般就是重点的 1 2 3，尤其是 JetPack 组件。

4 5 Android 源码，ART啥的一般用的不多，就是面试会假模假样的问问你。

要是 4 5 都有点研究了，你都可以出任移动架构师啦。

所以大家直接看 1 2 3 ，然后 4 5 去背面经高频题就中，不影响。

3. 开源库

1.网络请求（Retrofit2）

2.图片加载（Glide／Fresco)

3.事件总线（EventBus/Otto）

4.异步操作（RxJava2)

5.依赖注入（ButterKnife/Dagger2）

数据库操作，图片选择器，大图查看器

这些我们用的挺多的，因为用开源库用框架,比自己写方便多了。

不用专门去研究，就在下面做项目App的时候去多用这些开源库就会了。

然后面试的时候会假模假样的问问开源库源码的实现：

比如：Retrofit2 的底层实现逻辑是怎么样的？

一般是你简历上做的项目写了用了啥 开源库 框架，他就问这个开源库的底层实现。

还是不用自己研究，问法都固定。快面试的时候背面经就中。

4. 项目App

这个就随便做个差不多的 App 就中了。

一般有两个思路：

  1. 模仿主流APP；

  2. 自己的创意；

一般 1 简单点，拿微信举例子。就看微信是长什么样自己抄一个实现就中。

大家简历上比较多的都有 XX社交APP，XX电商APP，XX新闻，XX大学校园助手……

实在没思路直接 把《Android 开发入门课》里面的那几个小 Demo 拼凑以下也算一个了，

功能复杂度方面，自己再加加。找个实习应该中。

2 的话就需要自己有点想法了。这个没啥指导的，看个人创意。

要是 2 的话做好了可以上架酷安。或者一些手机厂商的应用商店。

然后有一定下载量，你就可以吹说自己的APP是多少人用，日活多少。然后后面优化怎么弄的。

基本这一套操作下来，基本BAT稳了。因为他们更喜欢这样的，怕就怕跟着团队做 App，然后队里大佬基本做好了，你简历上去挂个名。

比如B站一个朋友，他就是做的一个 霉霉 的专属 APP：有霉霉的一些事迹，歌曲，视频，还有很多粉丝的评论啥的。用户量挺大的。

你就可以来个 易烊千玺 APP啥的，在微博上一吆喝，用户量就有了，简历就好写了。

5.总结

作为一个 Android Boy，

如果比较急：

那就直接入门 -> 项目 App。

然后再项目中用下 3 中的开源库。

如果有点时间但不多：

那就 1 全吃，2中前 3 个学。然后就直接 做 项目 App + 用点开源库。然后在面试前把自己项目用到的 第三方开源库了解清楚。就可以直接上了。

如果时间很够：

2 中最后两项 和 3开源库源码这两个留着。其他全推。这两个在面试前背下相关的八股文就中。

其实Android Boy 在 Android 方面。虽然Android 的知识点很多，但是主要用的就那点。

学会基础的然后直接上手拼个项目，项目需要啥就去学什么。项目做好的还有时间就把 App优化一下。三方库啊，内存啊，网络啊。东西不多。

主要就是要这个能写简历上就中。

扩展：

Android Boy 其实还有一些技术是可以作为 加分项的：

比如 compose,Flutter,RN,混和开发。

嗯~，原生AndroidApp开发其实用的是 Kotlin + JetPack。

老项目是 Java + 三方库。

compose 写的挺方便但是用的不是很多。

Flutter，RN 是跨端的，学 Android 只能做 Android。但是现在公司总想着降低成本。

想一个前端/移动端直接把前面的东西全干了。

比如 Android，IOS，Web，PC，小程序。

前端可以直接用 H5 套娃，我们可以通过 Flutter和RN实现。

这是 原生 向跨端的迈进。

有个学姐就是学的 Android ，然后去美团转 RN 混合开发了。

但是市面上没有专门招 跨端什么 Flutter 开发。所以大家别搞错了。

Android 是基本盘，Compose，Flutter，RN 是加分项。

有能力和兴趣的可以试试。

客三消 应该是不会消的。人云亦云的人太多了，不仅暂时不会死反而是一个机会。

Android原生开发时一个需求不大但是不那么卷且最容易进大厂的最优解。
